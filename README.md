介绍
1. 语言概述
PomPov 是一门基于自定义汇编风格的底层编程语言，核心设计目标是简洁实现 “内存 / 磁盘操作 + 循环控制 + 资源管理”，语法兼具独特性与逻辑性，支持动态地址、批量操作和自动化资源回收，适用于底层配置、简单数值运算或小型工具开发。
2. 核心约定
2.1 大小写规则
指令、地址、操作符不区分大小写（如 s/S、P0300/p0300 等效），编译器会自动统一转为大写处理；
常量、标识串（如 CN3011SL1198731F）大小写敏感，需严格匹配定义。
2.2 地址类型
语言支持两类合法地址，遵循 “专属标识 + 合法格式” 规则：
地址类型	标识前缀	说明	合法示例
编辑器自建内存	P	由编译器动态创建，用于临时数据存储，支持补位和动态扩展	P0300、P2M81、P001X
系统磁盘	C:/D:	映射到操作系统合法路径（Windows：%LOCALAPPDATA%\PomPovEditor），持久化存储	C:CONFIG、D:DATA_01
2.3 语法符号
符号	作用	
:	分隔符，用于区分 “地址 - 操作数”“指令 - 参数”	
-	双重作用：1. 连接连续操作（如 pom-pom）；2. 绑定 “地址名 - 值”（如 P2MS0o-g）	
(/)	操作嵌套标记，从内到外执行，所有嵌套最终统一用一个 ] 收尾	
]	嵌套操作结束符，无论多少层嵌套，仅需一个 ] 闭合	
`	-]`	代码块结束标记（用于循环体收尾）
{g	常量绑定标记，用于关联循环终止常量	
3. 核心指令
3.1 基础操作指令（s/u）
3.1.1 设置指令 s（Set）
功能：初始化或覆盖目标地址的值（底层：直接赋值，而非累加）；
语法：s [目标地址] [操作组合]；
示例：
plaintext
s P0300 pov          ; 给内存地址P0300设置“加1”操作，直接赋值为初始值+1
s C:DATA pom-pom(pov] ; 给C盘地址C:DATA设置“减→减→加”操作，直接赋值为初始值-1
3.1.2 更新指令 u（Update）
功能：在目标地址当前值基础上累加操作结果（底层：增量更新）；
语法：u [可选模式] [目标地址/参数]；
模式说明：-a 表示算术模式（支持补位 + 偏移运算）；
示例：
plaintext
u -a -|s P0113:cxba 2|-]  ; 算术模式更新P0113，补2位+偏移2
u CN3011SL1198731F         ; 更新核心标识CN3011SL1198731F的值（累加1）
3.2 加减操作符（pom/pov）
操作符	功能	单操作效果	组合规则
pom	减	数值 - 1	支持连字符连接（pom-pom）和括号嵌套（pom(pov(pom]）
pov	加	数值 + 1	支持连字符连接（pov-pov）和括号嵌套（pov(pom(pov]）
组合操作解析规则
去除所有 (/-，保留操作符顺序；
按顺序累加操作效果（pom=-1，pov=+1）；
示例：
pom-pom(pov(pov(pom] → 解析为 pom+pom+pov+pov+pom → 计算：-1-1+1+1-1 = -1；
pov(pom-pov] → 解析为 pov+pom+pov → 计算：+1-1+1 = +1。
3.3 循环控制指令 kaj（Loop）
功能：实现条件循环，支持 “初始化 - 判断 - 终止” 全流程；
语法：kaj>:eaa [计数器地址] [操作组合]:eas [判断器地址] [操作组合]:{g [终止常量]:eau [终止标记地址]；
阶段标记说明：
:eaa：循环初始化阶段（配置计数器）；
:eas：循环条件判断阶段（配置判断器）；
:eau：循环终止准备阶段（配置终止标记）；
循环体格式：
plaintext
kaj>:eaa [计数器配置]:eas [判断器配置]:{g [终止常量]:eau [终止标记]
:cod -|  ; 循环体开始
    [循环执行的指令1]
    [循环执行的指令2]
|-]      ; 循环体结束
示例：
plaintext
kaj>:eaa P2M81 pov-pom:eas P2M82 pom-pom(pov(pom({g C3113941521]:eau P2M30
:cod -|
    u -a -|s P0113:cxba 2|-]
    s P001X pom-pom(pov(pov(pom]
|-]
循环终止条件：计数器地址（如 P2M81）的数值达到终止常量（如 C3113941521）时停止。
3.4 资源管理指令 sjxeaflist（Start/Shutdown List）
功能：程序启动时初始化参数，关机时自动回收资源（内存清空、磁盘文件删除）；
语法：sjxeaflist [参数1] [参数2] ...；
参数规则：
格式 1：地址名-值（绑定地址与初始值，值为字符时自动转为 ASCII 码）；
格式 2：纯地址（直接初始化，默认值为 0）；
示例：
plaintext
sjxeaflist P2MS0o-g P2M1H9  ; 绑定P2MS0o=g（ASCII值103），初始化P2M1H9=0
4. 关键语法特性
4.1 地址补位（cxba）
定义：cxba 是固定补位标记，默认补 2 位（可通过编译器配置修改）；
作用：给 P 开头内存地址补前导 0，确保地址格式统一，支持偏移运算；
语法：[地址]:cxba [偏移量]；
示例：
plaintext
s P0113:cxba 2  ; P0113补2位→P00113，加偏移量2→最终操作地址P00115
4.2 动态地址（X 通配符）
定义：P 开头地址中的 X 为通配符，自动扩展为 0-9 十个连续地址；
作用：实现批量地址操作，无需逐个编写指令；
示例：
plaintext
s P001X pom-pom(pov(pov(pom]  ; 扩展为P0010~P0019，批量执行“净减1”操作
4.3 常量绑定（{g）
功能：关联循环终止常量，用于控制循环次数或终止条件；
语法：{g [常量值]；
说明：常量值为整数，支持超长数字（如 C3113941521，前缀 C 为常量标识，可自定义）；
示例：
plaintext
{g C3113941521  ; 绑定终止常量3113941521
5. 完整程序结构示例
plaintext
; 1. 初始化普通内存地址
s P0300 pov
s P0314 pov
s P1209 pom-pom

; 2. 循环控制（初始化-判断-终止）
kaj>:eaa P2M81 pov-pom:eas P2M82 pom-pom(pov(pom({g C3113941521]:eau P2M30
:cod -|  ; 循环体开始
    ; 算术模式更新地址（补2位+偏移2）
    u -a -|s P0113:cxba 2|-]
    ; 动态地址批量操作
    s P001X pom-pom(pov(pov(pom]
    ; 更新核心标识
    u CN3011SL1198731F
|-]  ; 循环体结束

; 3. 启动/关机参数配置
sjxeaflist P2MS0o-g P2M1H9
6. 执行流程说明
启动阶段：执行 sjxeaflist 初始化参数，创建磁盘目录和内存空间；
初始化阶段：执行 s 指令设置普通地址的初始值；
循环阶段：
初始化计数器、判断器，绑定终止常量；
循环体按指令执行（动态地址批量操作、算术更新、核心标识刷新）；
每次循环后检查计数器数值，达到终止常量则停止循环；
关机阶段：自动清空编辑器内存，删除磁盘配置文件，释放资源。
7. 注意事项
嵌套操作必须以 ] 收尾，否则编译器会判定语法错误；
磁盘地址的路径由编译器自动管理，无需手动创建目录；
动态地址仅支持 P 开头内存地址，不支持磁盘地址；
循环终止常量需为整数，若输入非数值会导致程序异常终止；
补位仅对 P 开头地址生效，磁盘地址不参与补位运算。
